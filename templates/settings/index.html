What it is: the admin landing for everything that grants access to KubePosture — human users, RBAC groups, and cluster ingest tokens. Tabs above switch between them.

Where the data comes from: Django auth (users, groups) and the core.IngestToken table (one token per cluster, hashed at rest). New rows are created here, not by the import pipeline.

How to use: grant new humans the viewer / operator / admin group; mint one ingest token per cluster on first install and configure the import CronJob with it.
