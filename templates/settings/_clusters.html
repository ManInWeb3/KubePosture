Cluster Exposure Settings

These flags feed the effective priority decision tree. A CRITICAL CVE on an internet-exposed production cluster gets Immediate priority, while the same CVE on an isolated dev cluster gets Scheduled.
  • Internet Exposed — cluster has services reachable from the public internet (ingress, LoadBalancer). Increases priority for findings on this cluster.
  • Sensitive Data — cluster processes PII, financial, or regulated data. Findings on HIGH/MEDIUM severity get elevated to Scheduled instead of Defer.
  • Namespace Overrides — for mixed clusters where some namespaces are exposed and others are internal. Overrides inherit cluster defaults when not set.
  • Environment — auto-detected from cluster name on first ingest. Production clusters get higher priority than dev/staging.
