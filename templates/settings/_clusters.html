Clusters are auto-registered on first ingest — when the kubeposture-import CronJob POSTs scan data with a new cluster name, a row appears here. You never create clusters manually.

What to set: after a cluster appears, edit it to mark exposure flags — these feed the effective-priority decision tree. A CRITICAL CVE on an internet-exposed production cluster becomes Immediate; the same CVE on an isolated dev cluster becomes Scheduled.

  • Internet Exposed — services reachable from the public internet (ingress, LoadBalancer).
  • Sensitive Data — processes PII, financial, or regulated data. HIGH/MEDIUM findings escalate from Defer to Scheduled.
  • Namespace Overrides — for mixed clusters where only some namespaces are exposed.
  • Environment — auto-detected from cluster name; production outranks dev/staging.
