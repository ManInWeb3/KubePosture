What it is: a single workload's posture across every cluster it runs in — image rollout history, finding counts, deploy events, and per-image CVE list. The cluster picker scopes the page to one cluster; the trend chart marks image-set changes (deploy / rollback / admission rewrite) so you can see whether a change moved the needle on findings.

Where the data comes from: Trivy CRDs (vulnerabilities, config, secrets) and Kyverno PolicyReports ingested via webhook + the kubeposture-import CronJob. Snapshot rows feed the trend chart; the image table comes from inventory observations.

How to use: click an image row to load its active findings below, then click any finding to slide in its full detail panel (description, remediation, references). Use the trend toggles to flip between severity / urgency stacking and to hide heartbeat snapshots if you only want real deploy events.
